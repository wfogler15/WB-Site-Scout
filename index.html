Your pipeline is stored on the shared team backend. Every change syncs automatically and teammates see it live.

Pipeline backup

Saves a snapshot of every site, parcel, and note to a file on your computer. The backend already keeps your data safe — this is an extra belt-and-suspenders copy you can archive or send to restore from.

Parcel tiles · advanced

Denver parcels already load from the team's hosted tiles by default — you don't need to touch this. Only set a URL here to point at a different tiles file (e.g. after rebuilding the pipeline). Stored only in this browser.

Parcel cache

Parcels you load are saved in this browser so revisited areas repaint instantly — a big speed win on the slower Denver counties. Clear it to force fresh data if a county updates its records.

Add zoning from a file

For any jurisdiction without a live service, download its zoning as GeoJSON or a zipped shapefile from the city's open-data portal and load it here. It's stored in your browser and works offline, painting and resolving on click like the live cities.

Live zoning sources

Zoning is pulled from each city's own GIS. Currently wired: Phoenix, Mesa, Tempe, Gilbert, Scottsdale, Chandler. Click any parcel for the code and its source; the on-map readout reports what each city's server returned.
